# v0.1.0 - Initial Release

**Release Date:** 2025-02-04

The first public release of Pico-Boot, the zero-configuration bootstrap layer for the Pico ecosystem.

---

## Features

### Plugin Auto-Discovery
- Automatically discovers and loads modules from installed packages via Python entry points
- Uses the `pico_boot.modules` entry point group
- Graceful handling of plugin load failures (logs warning, continues)

### Custom Scanner Harvesting
- Collects `PICO_SCANNERS` from all loaded modules
- Merges with user-provided `custom_scanners`
- Preserves order: user-provided scanners first, then harvested

### Drop-in Replacement
- `pico_boot.init()` is a drop-in replacement for `pico_ioc.init()`
- Maintains full signature compatibility
- All `pico_ioc.init()` parameters are supported

---

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `PICO_BOOT_AUTO_PLUGINS` | `true` | Enable/disable plugin auto-discovery |

---

## Compatibility

- **Python:** 3.11, 3.12, 3.13, 3.14
- **pico-ioc:** >= 2.2.0

---

## Ecosystem Support

This release supports auto-discovery of:
- pico-fastapi
- pico-sqlalchemy
- pico-celery
- pico-pydantic
